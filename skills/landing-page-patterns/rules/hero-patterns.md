# Hero Section Patterns -- 8 Proven Layouts with Conversion Data

## Why the Hero Matters

The hero section is the first thing visitors see. 55% of visitors spend less than 15 seconds on a page. If the hero doesn't hook them, nothing below matters. Every hero pattern below has been tested across thousands of landing pages.

## Pattern 1: Text-Left, Product-Right (The SaaS Standard)

```
┌─────────────────────────────────────────┐
│  [Headline]           [Product          │
│  [Subheadline]         Screenshot       │
│  [CTA Button]          or Demo]         │
│  [Trust badges]                         │
└─────────────────────────────────────────┘
```

**When to use**: SaaS products with a visual interface. The product screenshot provides instant credibility.

**Conversion data**: 3-8% CVR for free trial, 12-20% CVR for lead gen.

**Best practices**:
- Product screenshot should show the actual UI, not a mockup
- Show the product in a "success state" (dashboard with good metrics, completed project)
- Add subtle animation to the product (cursor moving, data updating) for engagement
- On mobile: stack text above product screenshot

**Example headline**: "Ship Smart Contracts 10x Faster" with a screenshot of the IDE.

## Pattern 2: Centered Text with Background (The Bold Statement)

```
┌─────────────────────────────────────────┐
│                                         │
│         [Large Headline]                │
│       [Supporting subtext]              │
│         [CTA Button]                    │
│                                         │
│  [Background: gradient, image, or video]│
└─────────────────────────────────────────┘
```

**When to use**: Strong brand, emotional product, or when you don't have a product UI to show. Works well for B2C, communities, and events.

**Conversion data**: 5-15% CVR for email capture, 2-5% for paid signup.

**Best practices**:
- Headline must be punchy (5-8 words maximum)
- Background image/gradient must not compete with text (use overlay)
- Text contrast ratio must be at least 4.5:1 for accessibility
- CTA button should be high-contrast against the background
- On mobile: reduce headline size, increase padding

**Example headline**: "Build the Future of Finance" on a dark gradient with subtle code animation.

## Pattern 3: Video Hero (The Demo Hook)

```
┌─────────────────────────────────────────┐
│  [Headline]                             │
│  [Subheadline]                          │
│                                         │
│  ┌───────────────────────────────┐      │
│  │     [Video Player]            │      │
│  │     with play button          │      │
│  └───────────────────────────────┘      │
│                                         │
│  [CTA Button below video]               │
└─────────────────────────────────────────┘
```

**When to use**: Complex products that need demonstration, educational platforms, products where "seeing is believing."

**Conversion data**: 86% of marketers say video increases conversions. 2-4x engagement vs static image. Average lift: 20-30% over static hero.

**Best practices**:
- Thumbnail must be compelling (custom, not auto-generated)
- Video length: 60-90 seconds maximum for hero placement
- Auto-play with mute is acceptable (with clear unmute button)
- Never auto-play with sound (users will bounce immediately)
- Include text CTA below video for users who don't watch
- Video should show the product in action, not a talking head

## Pattern 4: Social Proof Hero (The Crowd)

```
┌─────────────────────────────────────────┐
│  "Join 4,500+ developers learning Web3" │
│                                         │
│  [Row of user avatars/photos]           │
│  [Star rating] "Rated 4.8/5 by 342     │
│   developers"                           │
│                                         │
│  [CTA Button: "Join for Free"]          │
│                                         │
│  [Logos: companies where users work]    │
└─────────────────────────────────────────┘
```

**When to use**: Products with strong community, high user count, good reviews. Works when you can prove popularity.

**Conversion data**: Social proof in hero increases conversion 15-20% vs no social proof. User count above 1,000 is the credibility threshold.

**Best practices**:
- Real photos, real numbers (never fake it)
- Specific numbers beat round numbers ("4,537 developers" > "4,500+ developers")
- Show avatars of real users (with permission) or aggregate photos
- Include logos of recognizable companies where users work
- If you have <1,000 users, lead with a different hero pattern and put social proof lower

## Pattern 5: Problem-Agitation Hero (The Pain Point)

```
┌─────────────────────────────────────────┐
│  "Tired of [painful thing]?"            │
│                                         │
│  [Illustration showing the problem]     │
│                                         │
│  "[Product] eliminates [problem]        │
│   so you can [desired outcome]"         │
│                                         │
│  [CTA: "Start solving this"]            │
└─────────────────────────────────────────┘
```

**When to use**: When your audience has a strong, specific pain point they recognize immediately. B2B, productivity tools, problem-solving products.

**Conversion data**: Pain-focused headlines convert 20-30% better than benefit-focused headlines for high-pain-awareness audiences.

**Best practices**:
- The problem must be something visitors recognize in the first 2 seconds
- Don't dwell on the problem for more than 2 lines (agitate, don't depress)
- Transition quickly to the solution (your product)
- CTA should be solution-oriented, not problem-oriented
- Test: ask 5 people in your target audience "does this describe you?" -- if <4 say yes, choose a different problem

## Pattern 6: Interactive/Calculator Hero (The Tool)

```
┌─────────────────────────────────────────┐
│  "Calculate Your Potential Savings"     │
│                                         │
│  ┌───────────────────────────────┐      │
│  │ Team size: [slider/input]     │      │
│  │ Current tool: [dropdown]      │      │
│  │ Monthly spend: [input]        │      │
│  │                               │      │
│  │ "You could save $X,XXX/month" │      │
│  └───────────────────────────────┘      │
│                                         │
│  [CTA: "Get Your Custom Plan"]          │
└─────────────────────────────────────────┘
```

**When to use**: Products with quantifiable ROI, cost savings, or personalized value. B2B SaaS, financial tools, infrastructure products.

**Conversion data**: Interactive elements increase time on page 2-3x and conversion 15-40% for B2B. The personalization creates investment.

**Best practices**:
- Maximum 3 input fields (more than 3 = friction)
- Show the result instantly (no "submit" button to see results)
- The result should be impressive (if your savings story is weak, don't use this pattern)
- CTA appears after result is shown (they're now invested)
- Save the inputs as lead data even before CTA click

## Pattern 7: Before/After Hero (The Transformation)

```
┌─────────────────────────────────────────┐
│  ┌──────────────┐  ┌──────────────┐    │
│  │   BEFORE     │  │   AFTER      │    │
│  │ [messy code] │→ │ [clean code] │    │
│  │ [slow]       │  │ [fast]       │    │
│  │ [confused]   │  │ [confident]  │    │
│  └──────────────┘  └──────────────┘    │
│                                         │
│  [Headline: "From [before] to [after]  │
│   in [timeframe]"]                      │
│  [CTA Button]                           │
└─────────────────────────────────────────┘
```

**When to use**: Education, transformation products, coaching, any product where the user undergoes a change.

**Conversion data**: Before/after comparisons increase perceived value 30-50%. Works exceptionally well for education and coaching.

**Best practices**:
- Before state must be relatable (visitor sees themselves)
- After state must be aspirational but believable
- Include a timeframe (makes the transformation concrete)
- Visual before/after is more powerful than text-only
- Use real examples, not hypothetical ones

## Pattern 8: Minimal Hero (The Confidence Play)

```
┌─────────────────────────────────────────┐
│                                         │
│                                         │
│         [Short, Bold Headline]          │
│         [CTA Button]                    │
│                                         │
│                                         │
└─────────────────────────────────────────┘
```

**When to use**: Well-known brands, products that don't need explanation, returning-visitor landing pages, premium/luxury positioning.

**Conversion data**: 2-5% CVR for signups. Lower than feature-rich heroes for unknown brands. Higher for established brands (Apple, Stripe, Linear).

**Best practices**:
- Only works if your brand is already known or the value prop is instantly clear
- Headline must be extremely strong (carries all the weight)
- Massive whitespace communicates confidence and premium quality
- Single CTA, no distractions
- Don't use this for paid traffic from cold audiences (they need more context)

## Hero Selection Decision Tree

```
Is your product visually impressive (has a UI)?
├── Yes → Pattern 1 (Product-Right) or Pattern 7 (Before/After)
└── No
    ├── Do you have strong social proof (1000+ users)?
    │   ├── Yes → Pattern 4 (Social Proof)
    │   └── No
    │       ├── Does your audience have a strong, specific pain point?
    │       │   ├── Yes → Pattern 5 (Problem-Agitation)
    │       │   └── No
    │       │       ├── Can you demonstrate clear ROI/savings?
    │       │       │   ├── Yes → Pattern 6 (Calculator)
    │       │       │   └── No → Pattern 2 (Centered) or Pattern 3 (Video)
    │       │       └──
    │       └──
    └──
```

## Universal Hero Rules

1. **One CTA only** in the hero. Never two equal-weight buttons above the fold.
2. **Headline under 10 words.** Every word must earn its place.
3. **Subheadline under 25 words.** Supports the headline, doesn't repeat it.
4. **CTA visible without scrolling** on both desktop and mobile.
5. **Load in under 2 seconds.** Hero images must be optimized (WebP, <200KB).
6. **Test on 375px width** before launching. 60%+ of paid traffic is mobile.
